# EM_Onboarding-Copy — オンボーディング文言集

> 初回起動時の文言。**ここで「またダメだった」と感じさせない**ことが最重要。
> 当事者は過去に「専門アプリ」「療育アプリ」で何度も傷ついてきている。

## ⭐ 設計方針

1. **「ディスレクシア」「障害」「症状」を1度も使わない**
2. **「あなたを治す」感を一切出さない**
3. 「楽しい」「便利」「自分らしい」を前面に
4. 「3問だけ」「30秒」など**短く終わる**ことを最初に伝える
5. **本人の選択**で進む（「次へ」ボタンに「お任せします」も併設）

## 画面別文言

### Screen 1 — Welcome

```
ようこそ。

文字を、あなたに合う見え方に変えてみませんか？

(続ける)  (今すぐ使う)
```

> **ポイント**：「文字を、あなたに合う見え方に」と機能を主語に。「あなたを支援する」ではなく「文字側を変える」フレーミング。

### Screen 2 — オンボーディング3問テスト 導入

```
3問だけ、読み比べテストをします。

正解はありません。
「こっちの方がスッと読める」を選んでください。

30秒で終わります。

(始める)  (スキップして自分で設定)
```

> **ポイント**：「正解はありません」が決定的に重要。診断や評価ではないことを明示。

### Screen 3 — 質問1（フォント）

```
Q1. どちらが読みやすそうですか？

[サンプルA] UDデジタル教科書体
[サンプルB] 明朝体

(A)  (B)  (どちらでもない)
```

> 「どちらでもない」を必ず用意。「選べない＝失敗」と感じさせない。

### Screen 4 — 質問2（背景色）

```
Q2. どちらの背景色が落ち着きますか？

[サンプルA] 白い背景
[サンプルB] クリーム色の背景

(A)  (B)  (どちらでもない)
```

### Screen 5 — 質問3（行間）

```
Q3. どちらが目で追いやすいですか？

[サンプルA] 普通の行間
[サンプルB] 広めの行間

(A)  (B)  (どちらでもない)
```

### Screen 6 — 結果提示

```
あなたの初期設定が見つかりました。

→ UDデジタル教科書体
→ クリーム色の背景
→ 広めの行間

いつでもサイドメニューから変えられます。

(この設定で始める)  (もう少しカスタマイズする)
```

> **ポイント**：「治療法が見つかりました」「あなたの障害に合う設定です」のような医療フレーミング**絶対回避**。「初期設定」「いつでも変えられる」を強調。

### Screen 7 — 強み紹介（オプション、ON/OFF可能）

```
（オフ可能）

ちなみに——

文字の読み書きが苦手な人は、こんな力を持つことが多いそうです：

・全体像を見渡す視覚的な思考
・新しいアイデアを生む発想力
・人を引き込む話し方
（出典：Made by Dyslexia / Randstad Intelligence 5.0）

(なるほど)  (今は見ない)
```

> **ポイント**：押し付けない。「そうです」「多い」と断言を避ける。引用を明示してエビデンス感を出す。ON/OFF可能。

### Screen 8 — 完了

```
準備ができました。

読みたいテキストを貼り付けるか、
ファイルをドロップしてください。

(始める)
```

## エラー時／質問テスト中断時

```
途中で終わっても大丈夫です。

設定は後からいつでも変えられます。

(やめる)  (続ける)
```

## 「気が向いたら」案内文

オンボーディングをスキップした人に、操作中にさりげなく：

```
（控えめなツールチップ）

3問テストをまだやっていません。
試すと自分にぴったりの設定が見つかるかもしれません。

(やってみる)  (いま忙しい)
```

> 何度も出さない。1度スキップしたら最低1週間は再表示しない。
