# EM_Error-Empty-States — エラー・空状態の声かけ

> **エラーは「失敗」ではない**。代替を即示し、責めず、ユーザーを行き詰まらせない。

## 設計原則：3要素

1. **何が起きたか**を端的に（責めない）
2. **代替方法**を即示す
3. 必要なら**詳細を任意で開ける**

## エラー画面の文言集

### ファイル読み込み失敗
```
このファイルは読めなかったみたい。

別のファイル形式（.txt / .md）を試してみてください。
または、テキストを直接貼り付けることもできます。

(別のファイル)  (テキストを貼り付ける)
```

### 読み上げが使えない（ブラウザ非対応）
```
お使いのブラウザでは読み上げが使えません。

Chrome / Edge / Safari の最新版だと使えるようになります。
他の機能（フォント・色・ふりがな）はそのまま使えます。

(続ける)
```
> 「あなたの環境がダメ」と感じさせない。「他はそのまま使える」を必ず添える。

### ふりがなが付かない（kuromoji読み込み失敗）
```
ふりがな機能の準備に失敗しました。

通信環境を確認して、もう一度試してみてください。
読み上げや色変更は今でも使えます。

(もう一度)  (一旦そのまま使う)
```

### オフラインで未キャッシュフォント
```
このフォントはオフラインでは使えません。

別のフォントを選ぶか、ネットに繋がってからもう一度試してください。

(別のフォントを選ぶ)
```

### 設定保存失敗（localStorage容量超過など）
```
設定の保存ができませんでした。

ブラウザのストレージがいっぱいかもしれません。
古い設定を整理してみてください。

(整理する)  (一旦このまま使う)
```

### 想定外エラー（フォールバック）
```
うまく動かなかったみたい。

ページを再読み込みしてみてください。
読み書きしていたテキストは消えません。

(再読み込み)
```
> 「Internal Server Error」「Error 500」のような技術用語は出さない。

## 空状態（Empty States）の文言集

### テキスト未入力
```
ここに読みたいテキストを貼り付けてください。

または、左下の📂からファイルを開けます。

例：「お知らせのプリント」「ニュース記事」など。
```
> 「使い方が分からないあなたへ」式に上から目線にしない。

### 保存済みプロファイルなし
```
まだ保存した設定はありません。

気に入った設定が見つかったら、💾から保存できます。
```

### 検索結果ゼロ
```
見つかりませんでした。

別の言葉で探してみてください。
```
> 「あなたの検索キーワードが」と責めない。

## 確認ダイアログ

### 設定リセット確認
```
最初の状態に戻しますか？

今の設定は消えますが、
オンボーディングの3問テストはやり直せます。

(戻す)  (やめる)
```

### テキストクリア確認
```
今のテキストを消しますか？

(消す)  (やめる)
```
> 「本当に？」「念のため確認」のような大袈裟さは避ける。

## ローディング表示

```
（控えめなスピナー）
読み込み中…
```

長時間ローディングが続く場合：
```
少し時間がかかっています…

ネットが遅いか、ファイルが大きいのかも。
あと少しで終わります。

(やめる)
```

## アクセシビリティ警告（必要な場合のみ）

例：超高速読み上げ（350wpm超）を設定しようとした時
```
かなり速い設定です。

研究では、350文字/分を超えると内容が頭に入りにくくなることが分かっています。

(このまま設定)  (もう少し遅くする)
```
> 「ダメです」と止めず、**選択肢を残す**。エビデンスを示し、選択を尊重。
