# I_User-Testing — 当事者テスト設計

## テストの目的

- 「**研究上のエビデンスはあるが、実際にこの生徒に効くか**」を確認
- プリセットの妥当性を当事者の主観評価で検証
- マイクロコピー（[[Empowering-Messages-MOC]]）が「傷つけない／救う」か検証

## テスト連携先候補

| 連携先 | 強み |
|---|---|
| **NPO EDGE** | 藤堂栄子氏のネットワーク、LSA講座修了者多数 |
| **発達性ディスレクシア研究会** | 言語聴覚士・心理士による専門レビュー |
| **東大DO-IT Japan**（[[F_UTokyo-Projects]]） | スカラー（当事者中高生・大学生）多数 |
| **触るグリフ SNSコミュニティ** | 大人当事者の体験談多い |
| **京都府総合教育センター**（[[F_Schools]]） | タブレット活用実績、通級教室との連携 |
| **東京学芸大附属小金井小学校（鈴木秀樹）** | 通常学級1人1台ICT環境 |
| **ディスレクシア月間（10月）** | オフライン体験会の機会 |

## テスト段階

### Stage 1：紙プロト（α前）
- 主要画面（オンボーディング・メイン画面・設定パネル）を Figma 等で
- 当事者2〜3名にウォークスルー
- 主に「使いたいか」「圧迫感がないか」を確認

### Stage 2：機能単位プロトタイプ（α）
- HTML/JS 単機能ずつ
  - フォント切替
  - 読み上げ＋同期ハイライト
  - 行ハイライト
- 各機能を個別に評価
- A/B テスト（プリセットの妥当性）

### Stage 3：統合MVP（β）
- 5〜10名で1〜2週間使用
- 使用ログ（オプトイン）で行動分析
- 半構造化インタビュー

### Stage 4：実環境テスト（GA前）
- 学校（通級・通常学級）で3〜4週間
- 教員からのフィードバック
- 印刷出力・合理的配慮申請の実用性

## 評価項目

### 客観指標
- 読み速度（音読 / 黙読、ベースラインvs最適設定）
- 読解正答率（簡単な内容質問）
- タスク完了時間（オンボーディング・設定変更）

### 主観指標
- SUS（System Usability Scale）
- 「読みやすさ」の自己評価（5段階）
- 「使い続けたいか」（5段階）
- 「友達に勧めたいか」（NPS）

### ⭐ 心理的安全性チェック（差別化軸）
- 「アプリを使っていて自分を否定された気持ちになる瞬間はあったか」
- 「『治療されている』『直されている』感じはあったか」
- 「人前で開いて恥ずかしいUIはあったか」
- 「教師の『がんばれ』式の励ましを思い出す瞬間はあったか」

## 倫理配慮

- **本人の同意を最優先**：保護者経由でも本人意思を確認
- **プライバシー**：使用ログはオプトイン、デフォルトOFF
- **「治療」「診断」と称さない**：あくまで「読みやすさのカスタマイズツール」
- 報酬：時間に見合った謝礼（図書券等）
- 結果のフィードバック：当事者本人に「あなたの声がどう反映されたか」を説明
