# M4: 教室・家庭での実装

## 学習目標

- 通常学級／通級／家庭で実践できる具体的配慮を3つ以上挙げられる
- 「特別扱いに見えない」UD的支援を設計できる
- ICT活用の典型パターンを知る

## キーポイント

1. **配布物のレイアウト改善**は最も導入しやすく効果が大きい
2. **ICT活用**（読み上げ、OCR、音声入力）が現場の標準になりつつある
3. **東京学芸大附属小金井小（鈴木秀樹）**の「**全員1人1台**」モデルが理想
4. 「特別扱いに見えない」ことが本人の心理的負担を減らす

## 1. 配布物のレイアウト改善（即実践可能）

### 教員が今日からできる7つの変更

1. **フォントを UDゴシック / Arial に変える**（明朝体を避ける）
2. **本文を18ptに**（最低でも16pt）
3. **行間を1.5以上に**
4. **左揃え**にする（両端揃えは禁止）
5. **1行50字以内**にする
6. **強調は太字のみ**（下線・斜体・全大文字を避ける）
7. **白い紙ではなくクリーム色の厚紙**を使う

→ [[L_Layout]] / [[G_Guidelines]] の知見の直接応用

### Word での実装

```
1. 「フォント」を BIZ UDPゴシック / Arial に
2. 「フォントサイズ」を 18pt に
3. 「段落」→「行間」を 1.5 に
4. 「段落」→「配置」を 左揃え に
5. 「ページ設定」→「余白」を広めに
```

## 2. ICT活用の典型パターン

### iPad（小学校で多い）
- VoiceOver / 画面読み上げ（標準）
- 音声入力（標準）
- テキスト認識表示（写真→OCR→読み上げ、標準）
- アプリ：ロイロノート、Keynote、Voice Dream Reader

### Chromebook（中高で増加）
- Google Workspace for Education
- ドキュメントの音声入力
- Read&Write 拡張
- スクリーンリーダー

### Windows + Microsoft 365
- **イマーシブリーダー**（リフロー＋音声＋ハイライト同期＋行集中）
- OneNoteによる板書写真貼付＋手書き＋音声録音
- ディクテーション

詳細は [[F_MEXT-NISE]]

## 3. ⭐ 東京学芸大附属小金井小学校の実践（鈴木秀樹教諭）

### 核心思想

> **「読み書きの困難さに気づかれない子も救われる、通常学級1人1台ICT環境」**

### 実践内容
- 文科省「インクルーシブ教育システム構築モデル事業」採択校
- 学習者用デジタル教科書の読み上げ・文字サイズ変更を**通常学級で全員に使用**
- 「ニコニコルーム」（畳・ボールプール・個別学習スペース）で休み時間に自由
- 大学院生がメンター役

### 教員にとっての含意
- **「LDの子だけ特別なツールを使う」モデルを脱却**
- 全員が ICT・デジタル教科書を使うことで、支援が**目立たない**
- 「あの子は特別」と思われない＝本人の心理的負担減

詳細は [[F_Schools]]

## 4. 通級指導教室での典型的活用

### ステップ式 DAISY活用
1. ハイライト見ながら聞く
2. 直後に音読
3. 音消してハイライトに合わせて音読

### アセスメント＋スキル指導
- 通級でアセスメント → 個別の方略指導
- AT（支援技術）の使い方を練習
- 通常学級で AT 利用での参加

詳細は [[F_Schools]] / [[F_Education-Boards]]

## 5. 家庭での実践

### 保護者が今日からできる5つのこと

1. **読み上げを「ずるい」と思わない**：聞いて理解することは「読書」と同じ価値
2. **音読を強制しない**：負担が大きく、自己嫌悪を強化する
3. **デジタルツールを「逃げ」ではなく「武器」と捉える**
4. **「がんばろう」「努力しよう」を控える**（→ [[TL_M2-Strength-Based]]）
5. **学校への合理的配慮申請を一緒に進める**（→ [[TL_M5-Reasonable-Accommodation]]）

## 6. 学習場面別の典型的配慮

### テスト
- 別室受験
- 時間延長（通常の1.5倍が目安）
- 問題文の読み上げ（教員 or ICT）
- 問題用紙の拡大印刷
- 解答方法の代替（ひらがな解答、PC入力、口頭試問）
- **黒い下敷きの使用**（読みやすさのため、テスト中もOK：NISE事例）

### 板書
- タブレットで撮影許可
- OneNote / Google Keep に貼付け
- 手書き or キーボードでメモ追記

### 作文
- 音声入力で下書き
- 箇条書きで構成
- 切り貼りで段落構成
- ふりがな・スペルチェックは ATで

### 教科書
- マルチメディアDAISY教科書（→ [[T_Japanese-Tools]]）
- AccessReading（Word/EPUB配信）
- UDブラウザ
- 撮影→OCR→読み上げ

## 7. このアプリで何ができるか

[[FT_MVP]] の機能を活用：

- **配布物の前**：教員が文章をアプリで開いて、生徒の好みに合わせてレイアウト確認
- **配布物の後**：生徒が文章を読みやすい設定で開く・読み上げる
- **テスト・宿題**：問題文をアプリで読み上げ、視認性カスタマイズ
- **作文の前**：参考資料を読みやすく
- **印刷**：配布物を生徒の設定で印刷（学年別ルビ付与など）

## 明日からできる3つのこと

1. **来週の配布資料を UDフォント・18pt・行間1.5・左揃え で作り直す**
2. **タブレットでの板書撮影と OCR→読み上げ を授業で1度試す**
3. **本アプリのURLを生徒に共有し、好きな設定を見つけてもらう**
