# FT_OCR — カメラ撮影＋OCR機能仕様

> **MVPに含まれる必須機能。**
> 学校の配布プリント・ノート・任意の印刷物をカメラで撮影し、OCRで抽出したテキストを本アプリの視認性カスタマイズ機能（フォント・字間・ルビ・読み上げ等）で読める形に変換する。

## 1. ユーザーストーリー

### US-1（最重要）：プリント読み上げ
> ディスレクシアの中学生Aさん。学校で配布された理科のプリントが読みにくい。
> Aさんはスマホでアプリを開き、プリントをカメラで撮影する。
> アプリが文字を抽出し、UDフォント・行間1.5・クリーム背景で表示。
> 「読み上げ」をタップすると、TTSが読み上げながら単語をハイライトする。

### US-2：教科書補完
> 小4のBさん。家でドリルをやりたいが漢字が読めない。
> 保護者がドリルのページを撮影。
> アプリが OCR で文字を抽出し、ルビ自動付与で読み上げ。

### US-3：板書代替
> 高校生Cさんは、板書を写すのが間に合わない。
> 友達のノートをスマホで撮影。
> OCR で文字起こし → 自分の設定で読みやすく表示。

## 2. 機能要件

### 2.1 入力
- [ ] スマホ：背面カメラ起動（`<input type="file" accept="image/*" capture="environment">`）
- [ ] PC：ファイル選択（ブラウザのファイルピッカー、ドラッグ＆ドロップ）
- [ ] 複数枚連続撮影（v1.1）

### 2.2 画像前処理
- [ ] 撮影画像のサムネイル表示
- [ ] 自動：grayscale 変換、コントラスト調整
- [ ] 手動：傾き補正（任意・スライダー）、トリミング（任意）
- [ ] 圧縮：長辺2000px以内（処理速度のため）

### 2.3 OCR 実行
- [ ] Tesseract.js v5+ を使用
- [ ] 言語：日本語（`jpn`）デフォルト、日英両対応（`jpn+eng`）
- [ ] 進捗インジケータ（「文字を読み取っています...」+ プログレスバー）
- [ ] 処理キャンセル可能

### 2.4 出力
- [ ] 抽出テキストをアプリ内エディタに表示
- [ ] 編集可能（OCR エラー修正用）
- [ ] 元画像は保存しない（プライバシー）
- [ ] 抽出テキストは localStorage に保存（任意）

### 2.5 視認性カスタマイズの統合
- [ ] OCR完了後、自動的に本アプリの視認性設定（フォント・字間・ルビ等）が適用される
- [ ] 「読み上げ」ボタンですぐにTTS開始可能

## 3. 非機能要件

### 3.1 パフォーマンス
- 1ページのプリント（500〜2000字程度）：**10秒以内**でOCR完了を目指す
- モバイル CPU で動作することを前提に最適化

### 3.2 オフライン動作
- [ ] **Tesseract.js のオフライン処理を優先**（画像をサーバーに送らない）
- [ ] 辞書ファイルは事前にキャッシュ（Service Worker、v1.1）

### 3.3 プライバシー
- [ ] **画像はサーバーに送信しない**
- [ ] 撮影画像は OCR 完了後、メモリから破棄
- [ ] 任意で「画像を保存」できる場合も localStorage（端末内）のみ

### 3.4 精度
- 印刷物の日本語：実用レベル（誤字率5%以下が目標）
- 手書き：v1.0 では対象外（v1.1 で API オプション）
- 縦書き：v1.0 では対象外（v1.1 で対応）

### 3.5 アクセシビリティ
- [ ] 撮影プロセスをスクリーンリーダーで利用可能
- [ ] 進捗状況を aria-live で通知
- [ ] エラー時の代替手段（「テキストを手入力する」リンク）

### 4.3 ⭐ マイクロコピー（[[EM_Tone-Principles]] 準拠）

- 「文字を読み取っています…」（NG：「OCR処理中」「文字認識実行中」）
- 「間違いがあれば直してね」（OCR精度を期待しすぎない誘導）
- 「うまく読み取れなかった場合：もう一度撮影／明るいところで／別の方法で」
- ❌「画像認識失敗」「OCR エラー」（技術用語）

## 5. エラーハンドリング

### 5.1 想定エラー

| ケース | 対応 |
|---|---|
| カメラ権限拒否 | 「ファイル選択」モードへ案内 |
| 画像が暗すぎる | 「明るいところで撮影してください」 |
| 文字が見つからない | 「文字が見つからなかったみたい。もう一度撮ってみよう」 |
| Tesseract 辞書未ロード | 「準備中です。もう少しお待ちください」 |
| メモリ不足（古いスマホ） | 「画像のサイズを小さくして再試行します」 |
| ブラウザ非対応 | 「お使いのブラウザでは使えないみたい。Chrome / Safari の最新版だと使えるよ」 |

### 5.2 エラー時の代替案
すべてのエラー画面で「**テキストを手入力する**」リンクを提供。

## 6. 実装ノート

詳細は [[I_OCR-Library]] に分離。

### 6.1 主要ライブラリ
- **tesseract.js v5+**（推奨：最新版）
- **辞書**：`tesseract.js-core`、日本語データ（`jpn.traineddata`）

### 6.2 Tesseract.js のセットアップ
```ts
import { createWorker } from 'tesseract.js';

const worker = await createWorker('jpn', 1, {
  logger: (m) => updateProgress(m.progress),
});

const { data: { text } } = await worker.recognize(imageBlob);
await worker.terminate();
```

### 6.3 画像前処理（Canvas API）
- グレースケール化：明度のみ抽出
- コントラスト調整：自動レベル補正
- 二値化（必要なら）：threshold=128

### 6.4 ファイル管理
- 撮影画像は `Blob` のまま保持、URL.createObjectURL で表示
- OCR 完了後は revokeObjectURL で破棄
- 抽出テキストのみアプリ内データに

## 8. テスト計画

### 8.1 撮影シナリオテスト
- [ ] 学校配布のプリント（白黒コピー）
- [ ] 教科書ページ（カラー）
- [ ] ノート（手書きは v1.1 で）
- [ ] スマホ画面のスクリーンショット
- [ ] 黒板の写真（教室照明下）

### 8.2 精度ベンチマーク
- 当事者テスターに「これは塾長のプリントです」と渡してテスト
- 誤読率を測定
- v1.0 リリース基準：500字のプリントで誤字15字以下（誤読率3%以下）

### 8.3 デバイス互換性
- [ ] iPhone Safari（iOS 15+）
- [ ] iPad Safari（iPadOS 15+）
- [ ] Android Chrome
- [ ] Windows Chrome / Edge
- [ ] Mac Safari / Chrome

## 9. 将来拡張（v1.1〜v2）

### v1.1
- 複数枚連続撮影 → 1つの文書として結合
- 手書き対応の API オプション（Google Cloud Vision、ユーザーがAPIキー持参）
- Service Worker で辞書事前キャッシュ（オフライン強化）

### v2
- 縦書き対応
- 表・図の構造保持（PDF出力など）
- 数式認識（MathJax 出力）
- 動画からのテキスト抽出（動画字幕代替）

## 10. 受け入れ基準（v1.0 リリース判定）

- [ ] スマホで撮影 → OCR → 読み上げまでが**3タップ以内**で実現
- [ ] 標準的な学校プリント（A4・印刷文字・1000字程度）が**10秒以内**にOCR完了
- [ ] OCR エラー時にも編集で修正可能
- [ ] 画像がサーバーに送信されていないことをネットワーク監視で確認
- [ ] スクリーンリーダーで利用可能（aria-live 通知）
- [ ] エラーメッセージがすべて [[EM_Tone-Principles]] に準拠
