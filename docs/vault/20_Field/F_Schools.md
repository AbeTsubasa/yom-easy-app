# F_Schools — 学校現場の実践

## ⭐東京学芸大附属小金井小学校（鈴木秀樹教諭）

本アプリ設計の**最重要参考事例**。

### 実践内容
- 文科省「インクルーシブ教育システム構築モデル事業」採択校（2013-2014）
- 2017年頃からICT本格活用
- 学習者用デジタル教科書の読み上げ・文字サイズ変更を**通常学級で全員に使用**させる
- 「ニコニコルーム」（畳・ボールプール・個別学習スペース）で休み時間に自由に過ごせる場
- 大学院生がメンター役

### 核心思想
> **「読み書きの困難さに気づかれない子も救われる、通常学級1人1台ICT環境」**

→ アプリ設計への含意：「ディスレクシア用」と見えない汎用UIで、すべての生徒が**自然に**使える形にする。「特別な支援を受けている」感を出さない。

## 京都府総合教育センター（ITEC）

- 2016〜2018年度、読み書き困難児にタブレット端末を無償貸与
- 通級指導教室での学習支援プロジェクト
- 府立特別支援学校のタブレット活用事例をデータベース化（「読みたい」「書きたい」のニーズで検索可能）

## 通常学級・通級での典型的な配慮

- 板書をタブレットで撮影→OneNote等に貼付け、上に手書き／キーボードでメモ
- 教科書をカメラ撮影→OCR→読み上げ
- テスト時：別室・時間延長・読み上げ・ひらがな解答・問題用紙拡大・黒い下敷き
- 通級と通常学級の役割分担：通級でアセスメント＋スキル指導＋AT練習、通常学級でAT利用での参加

## 観察された失敗パターン

- 「タブレットは特別なもの」と厳しく管理しすぎる学校では生徒のスキルが上達しにくい
- デジタル教科書を「特別扱い」と渋る学校現場の温度差（京都新聞報道）
- UDフォント単独導入では効果が出ない場合 → レイアウト・色とセットで提示が必要
- 音声教材の入手手続きが煩雑で、教員・保護者・本人が容易に使い始められない

## アプリ実装への示唆

- **「特別扱いに見えない」UI**：標準的なノートアプリやリーダー風の外観
- **教員ITリテラシー差を前提**：ワンタップで「読み上げモード」「カスタマイズモード」切替
- **印刷出力**：UDフォント・行間設定のまま印刷可能（教員ニーズ）
- **設定プロファイルの引き継ぎ**：年度替わり、端末変更でも継続できる
