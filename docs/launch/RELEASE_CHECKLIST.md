## Phase 1：クローズドβ（1〜2 週間）

**対象**：信頼関係のある生徒・保護者 5〜10 名
**目的**：致命的な不具合・引っかかる言葉を拾う

### 開始前

- [ ] [BETA_TESTERS.md](./BETA_TESTERS.md) を最新版で読み返した
- [ ] [FEEDBACK_FORM_TEMPLATE.md](./FEEDBACK_FORM_TEMPLATE.md) を元に Google Forms を作成済み
  - [ ] フォーム URL を取得済み
  - [ ] 匿名回答・メールアドレス収集 OFF
- [ ] テスター候補 5〜10 名を選定（年齢・読みづらさの種類のバラつきを意識）
- [ ] 各テスターさん（または保護者）に**口頭で**「これは塾長が作ったツール、データは外に出ない、いつでもやめてよい」を伝えた
- [ ] 13 歳未満のテスター候補がいる場合：**保護者に必ず同席依頼**
- [ ] 開始日・1 度目の感想聞き取り目安日を決めた

### 開始後 1〜2 週間

- [ ] LINE / メールで BETA_TESTERS.md の内容＋ URL ＋ Forms URL を送付
- [ ] 「いつでもやめていい」「返事も急がない」の姿勢を都度添える
- [ ] バグ報告が来たら、GitHub Issues に手元でメモ（テスターには技術詳細を求めない）

### Phase 1 → 2 の判断

- [ ] フィードバックを 1 回まとめて読んだ
- [ ] 致命的不具合（読めない・落ちる・誤った内容を表示する）はゼロになった
- [ ] 言葉の引っかかりを直した（あれば）
- [ ] テスターさんから「無理にもう一度試して」と感じさせる連絡を**していない**
- [ ] **次のフェーズに進む心の準備が、自分の側に整った**

## Phase 2：拡大β（2〜4 週間）

**対象**：専門家・支援団体経由のテスター 20〜50 名規模
**目的**：当事者層を広げて、合う／合わない の幅を見る

### 開始前

- [ ] Phase 1 のフィードバック反映済みの 0.9.x が公開済み
- [ ] 連携を依頼する団体を 2〜3 か所に絞った（例：NPO EDGE、発達性ディスレクシア研究会、地域の特別支援学校）
- [ ] 各団体への依頼文（「設計を見ていただけませんか」トーン）を準備済み
  - [ ] 「無料で・登録不要・データ収集なし」を 1 行目に
  - [ ] 「効果検証より、設計の率直なご意見をいただきたい」を明示
  - [ ] 強引な依頼にならない言葉選び
- [ ] 専門家向けには、技術詳細（GitHub URL、ACCESSIBILITY.md）も添える
- [ ] Phase 1 で使った Forms に「Phase 2 参加経緯」の任意項目を追加（団体経由か個人経由か）

### 開始後 2〜4 週間

- [ ] 返信が来たら、急がず順次対応
- [ ] 専門家から構造的なフィードバック（複数の項目で同じ指摘）が出たら、その項目を Phase 3 前に修正
- [ ] 拒絶や無視も「お返事不要」の姿勢で受け止める

### Phase 2 → 3 の判断

- [ ] 専門家から大きな致命的指摘がない
- [ ] アクセシビリティ手動チェック（[ACCESSIBILITY.md](../../ACCESSIBILITY.md) の手順）が済んでいる
- [ ] axe DevTools / WAVE で critical 0 を確認
- [ ] 公開告知の文面を準備済み（次項参照）

## Phase 3：一般公開

**対象**：日本国内のディスレクシアでお困りの方すべて
**目的**：必要な人に届く

### 公開前

- [ ] バージョンを 1.0.0 に上げるか、0.9.x のまま公開するかを決めた
- [ ] README.md の冒頭が公開向けに読みやすい
- [ ] 告知文の言葉選び（[BETA_TESTERS.md](./BETA_TESTERS.md) と同じトーン）
  - [ ] 「治す」「克服」「画期的」を一切使っていない
  - [ ] 「もし合うようなら」「試して、合わなければやめられる」を必ず添えた
  - [ ] 効果を約束していない
  - [ ] 当事者の写真や事例を勝手に使っていない
- [ ] 告知メディアを決めた（塾の公式 SNS / 塾長の note / 信頼できる支援団体に共有依頼）
- [ ] バグ報告窓口を案内（GitHub Issues、もしくは塾の問い合わせフォーム）
- [ ] **長期サポート方針を自分に確認**：月 1 回程度のアップデート、緊急パッチの体制

### 公開当日

- [ ] 告知ポスト
- [ ] 反応に過剰反応しない（褒められても、批判されても、深呼吸）
- [ ] **「使ったけどイマイチだった」声を歓迎する姿勢**を明示

### 公開後 1 ヶ月

- [ ] フィードバックを月 1 回まとめて見る
- [ ] 致命的バグは 1 週間以内にパッチ
- [ ] それ以外は月 1 のリリースサイクルへ

## いつでも：自分自身の心のために

塾長が燃え尽きると、このアプリは終わります。長く続けることが、結果としていちばん多くの方に届くことになります。

- [ ] 「すべてのフィードバックに返事」を自分に課さない
- [ ] 「すべての要望に応える」を自分に課さない
- [ ] 「批判は人格否定ではない」と区別して受け取る
- [ ] 月に 1 度、このアプリの存在を**忘れる週末**を作る
- [ ] 困ったら、Issues に書く（公開してもよい）か、ご家族・同業の方に話す
